## <a name="what-are-the-issues-when-you-deploy-a-membership-database"></a><span data-ttu-id="b6310-111">¿Cuáles son los problemas al implementar una base de datos de pertenencia?</span><span class="sxs-lookup"><span data-stu-id="b6310-111">What Are the Issues When You Deploy a Membership Database?</span></span>

<span data-ttu-id="b6310-112">En la mayoría de los casos, al diseñar una estrategia de implementación para una base de datos, lo primero que debe tener en cuenta es qué datos desea implementar.</span><span class="sxs-lookup"><span data-stu-id="b6310-112">In most cases, when you devise a deployment strategy for a database, the first thing you need to consider is what data you want to deploy.</span></span> <span data-ttu-id="b6310-113">En un entorno de desarrollo o prueba, puede implementar datos de cuentas de usuario para facilitar la prueba de manera rápida y sencilla.</span><span class="sxs-lookup"><span data-stu-id="b6310-113">In a development or test environment, you might want to deploy user account data to facilitate quick and easy testing.</span></span> <span data-ttu-id="b6310-114">En un entorno de ensayo o producción, es muy poco probable que desearía implementar datos de la cuenta de usuario.</span><span class="sxs-lookup"><span data-stu-id="b6310-114">In a staging or production environment, it's very unlikely that you'd want to deploy user account data.</span></span>

<span data-ttu-id="b6310-115">Lamentablemente, las bases de datos de pertenencia ASP.NET presentan algunos desafíos específicos que tomar esta decisión mucho más complejo:</span><span class="sxs-lookup"><span data-stu-id="b6310-115">Unfortunately, ASP.NET membership databases introduce some specific challenges that make this decision a lot more complex:</span></span>

- <span data-ttu-id="b6310-116">Implementación de una solo esquema dejará la base de datos de pertenencia en un estado no operativo.</span><span class="sxs-lookup"><span data-stu-id="b6310-116">A schema-only deployment will leave the membership database in a non-operational state.</span></span> <span data-ttu-id="b6310-117">Esto es porque la base de datos de pertenencia incluye algunos datos de configuración (en el **aspnet\_SchemaVersions** tabla) que requiere la base de datos para poder funcionar.</span><span class="sxs-lookup"><span data-stu-id="b6310-117">This is because the membership database includes some configuration data (in the **aspnet\_SchemaVersions** table) that the database requires in order to function.</span></span> <span data-ttu-id="b6310-118">Por lo tanto, si lleva a cabo una implementación solo de esquema de la base de datos de pertenencia con el fin de excluir los datos de la cuenta de usuario, deberá ejecutar un script posterior a la implementación para agregar los datos de la configuración básica.</span><span class="sxs-lookup"><span data-stu-id="b6310-118">As such, if you perform a schema-only deployment of your membership database in order to exclude user account data, you'll need to run a post-deployment script to add the essential configuration data.</span></span>
- <span data-ttu-id="b6310-119">Según cómo esté configurada la base de datos de pertenencia, el proveedor de pertenencia puede usar la clave del equipo para cifrar las contraseñas y almacenarlos en la base de datos.</span><span class="sxs-lookup"><span data-stu-id="b6310-119">Depending on how your membership database is configured, the membership provider may use the machine key to encrypt passwords and store them in the database.</span></span> <span data-ttu-id="b6310-120">En este caso, los datos de la cuenta de usuario que se implementa con la base de datos quedará inservible en el servidor de destino.</span><span class="sxs-lookup"><span data-stu-id="b6310-120">In this case, any user account data you deploy with the database will become unusable on the destination server.</span></span> <span data-ttu-id="b6310-121">Por este motivo, la implementación de datos de la cuenta de usuario no es un escenario admitido.</span><span class="sxs-lookup"><span data-stu-id="b6310-121">For this reason, deploying user account data is not a supported scenario.</span></span>

## <a name="choosing-a-membership-database-strategy"></a><span data-ttu-id="b6310-122">Elegir una estrategia de base de datos de pertenencia</span><span class="sxs-lookup"><span data-stu-id="b6310-122">Choosing a Membership Database Strategy</span></span>

<span data-ttu-id="b6310-123">Siga estas directrices para elegir cómo aprovisionar una base de datos de pertenencia en un entorno de servidor empresarial:</span><span class="sxs-lookup"><span data-stu-id="b6310-123">Use these guidelines when you choose how to provision a membership database in an enterprise server environment:</span></span>

- <span data-ttu-id="b6310-124">Siempre que sea posible, implemente las bases de datos de pertenencia.</span><span class="sxs-lookup"><span data-stu-id="b6310-124">Wherever possible, do not deploy membership databases.</span></span> <span data-ttu-id="b6310-125">En el servidor de base de datos de destino en su lugar, cree manualmente la base de datos de pertenencia.</span><span class="sxs-lookup"><span data-stu-id="b6310-125">Instead, create the membership database manually on the target database server.</span></span> <span data-ttu-id="b6310-126">Si no ha personalizado el esquema de base de datos de pertenencia, puede simplemente crear una nueva in situ en el destino mediante el [herramienta de registro de SQL Server de ASP.NET (aspnet\_regsql.exe)](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx).</span><span class="sxs-lookup"><span data-stu-id="b6310-126">If you haven't customized your membership database schema, you can simply create a new one in situ at the destination using the [ASP.NET SQL Server Registration Tool (aspnet\_regsql.exe)](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx).</span></span>
- <span data-ttu-id="b6310-127">Si no tiene ninguna opción, pero para implementar una base de datos de pertenencia&#x2014;por ejemplo, si ha realizado modificaciones extensas el esquema de base de datos&#x2014;debe realizar una implementación solo de esquema de la base de datos de pertenencia, para excluir los datos de la cuenta de usuario y, a continuación, ejecutar un script posterior a la implementación para agregar los datos de configuración necesarias.</span><span class="sxs-lookup"><span data-stu-id="b6310-127">If you have no option but to deploy a membership database&#x2014;for example, if you've made extensive modifications to the database schema&#x2014;you should perform a schema-only deployment of the membership database, to exclude user account data, and then run a post-deployment script to add any required configuration data.</span></span> <span data-ttu-id="b6310-128">Puede encontrar amplia información sobre estos enfoques en [Cómo: Implementar la base de datos de pertenencia ASP.NET sin necesidad de incluir cuentas de usuario](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx).</span><span class="sxs-lookup"><span data-stu-id="b6310-128">You can find broad guidance on these approaches in [How to: Deploy the ASP.NET Membership Database Without Including User Accounts](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx).</span></span>

<span data-ttu-id="b6310-129">Es importante recordar que *el esquema de la base de datos de pertenencia es probable que sea bastante estático*.</span><span class="sxs-lookup"><span data-stu-id="b6310-129">It's important to remember that *the schema of your membership database is likely to be fairly static*.</span></span> <span data-ttu-id="b6310-130">Incluso si ha personalizado la base de datos de pertenencia, es improbable que necesitará actualizar el esquema de forma periódica&#x2014;no va a cambiar con la misma frecuencia que el código en una aplicación web o un proyecto de base de datos.</span><span class="sxs-lookup"><span data-stu-id="b6310-130">Even if you've customized the membership database, it's unlikely that you'll need to update the schema on a regular basis&#x2014;it's not going to change with the same frequency as the code in a web application or a database project.</span></span> <span data-ttu-id="b6310-131">Por lo tanto, no es necesario incluir la base de datos de pertenencia en los procesos de implementación automatizada o paso a paso.</span><span class="sxs-lookup"><span data-stu-id="b6310-131">As such, you shouldn't need to include the membership database in any automated or single-step deployment processes.</span></span>

## <a name="using-vsdbcmd-to-update-a-membership-database-schema"></a><span data-ttu-id="b6310-132">Uso de VSDBCMD para actualizar un esquema de base de datos de pertenencia</span><span class="sxs-lookup"><span data-stu-id="b6310-132">Using VSDBCMD to Update a Membership Database Schema</span></span>

<span data-ttu-id="b6310-133">Si modifica la estructura de la base de datos de pertenencia tras la primera implementación, es posible que no desea usar la herramienta de implementación Web de Internet Information Services (IIS) (Web Deploy) para volver a implementar la base de datos.</span><span class="sxs-lookup"><span data-stu-id="b6310-133">If you modify the structure of your membership database after your first deployment, you may not want to use the Internet Information Services (IIS) Web Deployment Tool (Web Deploy) to redeploy the database.</span></span> <span data-ttu-id="b6310-134">La funcionalidad de implementación de la base de datos de Web Deploy no incluye la capacidad de realizar actualizaciones diferenciales de una base de datos de destino&#x2014;en su lugar, la Web Deploy debe quitar y volver a crear la base de datos.</span><span class="sxs-lookup"><span data-stu-id="b6310-134">The database deployment functionality in Web Deploy doesn't include the capability to make differential updates to a destination database&#x2014;instead, Web Deploy must drop and re-create the database.</span></span> <span data-ttu-id="b6310-135">Esto significa que perder los datos de cuenta de usuario existentes, que no es deseable normalmente en entornos de ensayo o producción.</span><span class="sxs-lookup"><span data-stu-id="b6310-135">This means that you lose any existing user account data, which is typically undesirable in staging or production environments.</span></span>

<span data-ttu-id="b6310-136">La alternativa es usar la utilidad VSDBCMD para actualizar el esquema de la base de datos de destino.</span><span class="sxs-lookup"><span data-stu-id="b6310-136">The alternative is to use the VSDBCMD utility to update the schema of your destination database.</span></span> <span data-ttu-id="b6310-137">VSDBCMD incluye dos funciones importantes.</span><span class="sxs-lookup"><span data-stu-id="b6310-137">VSDBCMD includes two important capabilities.</span></span> <span data-ttu-id="b6310-138">En primer lugar, puede importar el esquema de base de datos existente en un archivo .dbschema.</span><span class="sxs-lookup"><span data-stu-id="b6310-138">First, it can import the schema of an existing database into a .dbschema file.</span></span> <span data-ttu-id="b6310-139">En segundo lugar, puede implementar un archivo .dbschema a una base de datos existente como una actualización diferencial, lo que significa que solo realiza los cambios necesarios para la base de datos de destino al día y no perderá ningún dato.</span><span class="sxs-lookup"><span data-stu-id="b6310-139">Second, it can deploy a .dbschema file to an existing database as a differential update, which means that it only makes the changes required to bring the target database up to date and you don't lose any data.</span></span>

<span data-ttu-id="b6310-140">Puede usar estos pasos generales para actualizar un esquema de base de datos de pertenencia:</span><span class="sxs-lookup"><span data-stu-id="b6310-140">You can use these high-level steps to update a membership database schema:</span></span>

1. <span data-ttu-id="b6310-141">Utilice la herramienta VSDBCMD **importación** acción para generar un archivo .dbschema para la base de datos de pertenencia de origen.</span><span class="sxs-lookup"><span data-stu-id="b6310-141">Use the VSDBCMD **Import** action to generate a .dbschema file for your source membership database.</span></span> <span data-ttu-id="b6310-142">Este procedimiento se describe en [Cómo: Importar un esquema desde un símbolo del sistema](https://msdn.microsoft.com/library/dd172135.aspx).</span><span class="sxs-lookup"><span data-stu-id="b6310-142">This procedure is described in [How to: Import a Schema from a Command Prompt](https://msdn.microsoft.com/library/dd172135.aspx).</span></span>
2. <span data-ttu-id="b6310-143">Utilice la herramienta VSDBCMD **implementar** acción para implementar el archivo .dbschema en la base de datos de pertenencia de destino.</span><span class="sxs-lookup"><span data-stu-id="b6310-143">Use the VSDBCMD **Deploy** action to deploy the .dbschema file to your destination membership database.</span></span> <span data-ttu-id="b6310-144">Este procedimiento se describe en [referencia de línea de comandos de VSDBCMD. EXE (implementación e importación del esquema)](https://msdn.microsoft.com/library/dd193283.aspx).</span><span class="sxs-lookup"><span data-stu-id="b6310-144">This procedure is described in [Command-Line Reference for VSDBCMD.EXE (Deployment and Schema Import)](https://msdn.microsoft.com/library/dd193283.aspx).</span></span>
